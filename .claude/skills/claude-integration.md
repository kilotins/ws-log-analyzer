# Claude API Integration Patterns

## Prompt Structure

All prompts are built in `wslog.py` via `build_claude_prompt()`. The system prompt
enforces a 5-section response structure:

1. **What this usually means**
2. **Most likely causes**
3. **What to check next**
4. **Suggested Splunk searches** (each in its own ```spl block)
5. **Confidence / limitations**

Optional style modifiers (e.g., `SWEDISH_CHEF_STYLE`) are appended to the system prompt
via the `style=` parameter. Never modify the base `CLAUDE_SYSTEM_PROMPT` for style changes.

## Security (Prompt Injection Protection)

- `_sanitize_prompt_input()` strips XML delimiter tags and escapes entities
- User queries and log excerpts are wrapped in `<user_query>` / `<log_excerpt>` tags
- System prompt explicitly tells Claude to treat these as DATA, not instructions
- `redact()` runs on all event text before it reaches the prompt
- Never include raw unsanitized user input in the system prompt

## Caching

Cache keys are generated by `claude_cache_key()` based on:
- User query (lowercased, trimmed)
- Matched codes, exceptions, tags
- Match type (code/exception/tag/none)

**Do NOT include event text hashes** — same structural match should cache-hit
regardless of minor text variations between log parses.

Two cache layers:
1. **Session cache** (`st.session_state.claude_cache`) — in-memory, fastest
2. **File cache** (`cache/claude_responses.json`) — persists across sessions

On file cache hit, promote to session cache. File cache survives new analyses;
session state resets. "Clear Claude cache" button clears both.

Swedish Chef mode appends `:swedish_chef` to the cache key.

## API Call Pattern

```python
from anthropic import Anthropic
client = Anthropic(api_key=key)
message = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=2048,
    system=prompt["system"],
    messages=[{"role": "user", "content": prompt["user"]}],
)
```

## API Key Storage

Stored in macOS Keychain via `keyring.set_password("ws-log-analyzer", "anthropic_api_key", key)`.
Falls back to in-memory session state if keyring is unavailable.
